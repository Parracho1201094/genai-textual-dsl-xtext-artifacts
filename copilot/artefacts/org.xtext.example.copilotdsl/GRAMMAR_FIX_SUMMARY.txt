❌ THE PROBLEM (What You Encountered)
═══════════════════════════════════════════════════════════════════════════════

Error Message:
  [ERROR] Cannot change type twice within a rule 
          (ErrorCode: MoreThanOneTypeChangeInOneRule)

Affected Lines: 222, 225, 229, 237, 240, 246 in Lts.xtext

Root Cause:
  Xtext prohibits multiple alternatives in a rule from creating different 
  object types. The expression rules violated this constraint.

Examples of violations:
  ❌ OrExpression with ('or' | '||') - each creates different type
  ❌ ComparisonExpression with (= | != | in | like | between) - mixed types
  ❌ Literal with (String | Number | Boolean | Null) - inheritance pattern

📝 DETAILED CHANGES
═══════════════════════════════════════════════════════════════════════════════

BEFORE (Incorrect):
──────────────────
OrExpression:
  AndExpression (('or' | '||') AndExpression)*;

AndExpression:
  ComparisonExpression (('and' | '&&') ComparisonExpression)*;

ComparisonExpression:
  AdditiveExpression (
    (op=('=' | '!=') AdditiveExpression)
    | ('in' '(' literals+=Literal (',' literals+=Literal)* ')')  ← CONFLICT
    | ('like' pattern=STRING)                                    ← CONFLICT
    | ('between' min=Literal 'and' max=Literal)                  ← CONFLICT
  )*;

Literal:
  StringLiteral | NumberLiteral | BooleanLiteral | NullLiteral;  ← Multiple types


AFTER (Correct):
───────────────
OrExpression returns Expression:
  AndExpression (({Expression.left=current} op=('or' | '||')) 
  right=AndExpression)*;

AndExpression returns Expression:
  ComparisonExpression (({Expression.left=current} op=('and' | '&&')) 
  right=ComparisonExpression)*;

ComparisonExpression returns Expression:
  AdditiveExpression (
    (({Expression.left=current} op=('=' | '!=')) right=AdditiveExpression)
    | ({Expression.left=current} op='in' '(' literals+=Literal 
       (',' literals+=Literal)* ')')  ✓ Same type
    | ({Expression.left=current} op='like' pattern=STRING)  ✓ Same type
    | ({Expression.left=current} op='between' 'between' 
       min=Literal 'and' max=Literal)  ✓ Same type
  )*;

Literal:
  {Literal} (stringValue=STRING | numberValue=NUMBER | 
             boolValue=BoolValue | nullValue='null');  ✓ Single type

🔧 HOW IT WORKS NOW
═══════════════════════════════════════════════════════════════════════════════

ACTION SYNTAX: {ClassName.attribute=value}

Example: {Expression.left=current} op=('or' | '||')

This means:
  1. Create a new Expression object
  2. Assign the current (left side) object to its 'left' attribute
  3. Store the operator ('or' or '||') in the 'op' attribute
  4. The right side is captured in the 'right' attribute

Result:
  • Single Expression type for all operators
  • No type conflicts
  • Operator information preserved in attributes
  • Xtext constraint satisfied ✓
