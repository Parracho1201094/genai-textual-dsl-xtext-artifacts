# GenAI Textual DSL Engineering Artefacts

This repository contains the artefacts used in the dissertation *Evaluating the Impact of using Generative AI for the Creation of Textual DSLs*. The study evaluates how Generative AI tools can support the construction of textual Domain-Specific Languages in Eclipse Xtext, using the ENORM/LTS case study as the empirical basis.

The repository is intended to make the case study traceable by collecting the artefact sets analysed in the dissertation: the human-developed ENORM/LTS Xtext reference implementation and the artefacts produced through Codex, ChatGPT, GitHub Copilot, and Claude. These artefacts include grammars, example models, invalid models, validators, quick fixes, generators, runtime or generated-code material, documentation, prompts, correction notes, and retained execution or verification evidence where available.

The repository should not be interpreted as a set of production-ready DSL implementations. Some artefact sets are complete executable scaffolds, while others are starter packages, IDE-assisted workspaces, or repaired source packages. Their value lies in supporting the comparative analysis of artefact quality, toolchain operability, documentation reliability, human correction effort, and the risks of using LLMs in textual DSL engineering.

## Ethical and authorship note

The human-developed ENORM/LTS reference artefacts originate from previous DSL engineering coursework and are included only as technical comparison material, with permission for use within the dissertation context. The repository does not evaluate the original students, their grades, or their individual performance. The artefacts are used solely to support comparison with LLM-supported DSL engineering workflows.
